## 3. Task Boards & Kanban

### Creating a Board
1. Navigate to your workspace
2. Click "+ New Board" in the left sidebar
3. Choose a template (Kanban, Scrum, Custom) or start blank
4. Name your board and set visibility (private, team, public)

### Default Columns
- **To Do:** Tasks not yet started
- **In Progress:** Currently being worked on
- **Review/QA:** Ready for review or testing
- **Done:** Completed tasks

### Customizing Columns
You can create custom columns to match your workflow:
1. Click the "..." menu on any board
2. Select "Board Settings" → "Columns"
3. Add, rename, reorder, or delete columns
4. Set WIP (Work In Progress) limits per column
5. Configure column colors and icons

### Custom Fields
Add structured data to tasks:
- **Text:** Single-line or multi-line text
- **Number:** Numeric values (hours, story points)
- **Date:** Due dates, milestones
- **Dropdown:** Pre-defined options
- **Person:** Assign team members
- **Checkbox:** Binary yes/no fields

**Pro Tip:** Custom fields are available on Pro and Business plans.

## 4. Real-Time Docs & Wikis

### Creating Documents
1. Go to the "Docs" section in your workspace
2. Click "+ New Document"
3. Choose from templates (meeting notes, specs, runbooks) or blank
4. Start typing with our rich text editor

### Features
- **Real-time Collaboration:** Multiple people can edit simultaneously
- **Comments & Suggestions:** Leave feedback without changing content
- **Version History:** See who changed what and when; restore previous versions
- **Embeds:** Insert tasks, boards, images, videos, code snippets
- **AI Assist:** Generate summaries, improve writing, extract action items

### Wiki Structure
Organize documentation hierarchically:
- **Spaces:** Top-level categories (e.g., "Engineering", "Marketing")
- **Pages:** Individual documents within spaces
- **Sub-pages:** Nested documentation for detailed topics

### Permissions
Control who can view/edit documents:
- **Owner:** Full control, can delete
- **Editor:** Can edit and comment
- **Commenter:** Can only comment
- **Viewer:** Read-only access

## 5. Integrations

### Slack Integration
**Connect your workspace to Slack for:**
- Task notifications in channels
- Create tasks from Slack messages
- Daily standup summaries
- @mention notifications

**Setup:**
1. Go to Settings → Integrations → Slack
2. Click "Connect to Slack"
3. Authorize TechCorp in your Slack workspace
4. Configure which channels receive notifications

### Google Drive Integration
**Attach and preview files directly from Google Drive:**
- Link Drive files to tasks
- Preview documents without leaving TechCorp
- Automatic sync when files are updated

**Setup:**
1. Settings → Integrations → Google Drive
2. Sign in with your Google account
3. Grant TechCorp access to Drive

### GitHub Integration
**Available on Pro and Business plans**

**Features:**
- Link GitHub issues to TechCorp tasks
- Automatic status updates when PRs are merged
- Commit history visible on tasks
- Deploy notifications

**Setup:**
1. Settings → Integrations → GitHub
2. Click "Connect GitHub"
3. Authorize TechCorp to access your repositories
4. Select which repos to sync

**Troubleshooting GitHub Integration:**
- Ensure you have admin access to both TechCorp and GitHub
- Check that your GitHub organization allows third-party app access
- If you see "Authorization failed," disconnect and reconnect the integration
- For private repos, ensure the TechCorp app has been granted access

## 7. Team Workspaces & Permissions

### Workspace Roles
| Role | Permissions |
|------|-------------|
| **Owner** | Full control, billing, delete workspace |
| **Admin** | Manage members, settings, boards |
| **Member** | Create/edit tasks, docs, comments |
| **Guest** | Limited access to shared items only |

### Adding Team Members
1. Click workspace name → "Members"
2. Click "Invite Members"
3. Enter email addresses (comma-separated)
4. Choose role (Member, Admin, Guest)
5. Select which boards/docs they can access
6. Send invitation

### Guest Access
**Guests are external collaborators with limited permissions:**
- Can only see boards/docs explicitly shared with them
- Cannot invite other members
- Cannot access workspace settings
- Count: 5 guests per workspace (Pro), unlimited (Business)

### Permission Levels for Boards
- **Public (Workspace):** All workspace members can view
- **Team:** Only specific team members can view
- **Private:** Only explicitly added members can view

## 9. API & Developer Resources

### REST API
**Base URL:** `https://api.techcorp.com/v1`

**Authentication:**
- API keys (Settings → Developer → API Keys)
- OAuth 2.0 for third-party apps

**Rate Limits:**
| Plan | Requests/minute | Requests/day |
|------|-----------------|--------------|
| Free | 60 | 1,000 |
| Pro | 300 | 10,000 |
| Business | 1,000 | 100,000 |
| Enterprise | Custom | Custom |

**Key Endpoints:**
- `GET /workspaces` - List workspaces
- `GET /boards` - List boards in workspace
- `GET /tasks` - List tasks on a board
- `POST /tasks` - Create a new task
- `PUT /tasks/{id}` - Update a task
- `DELETE /tasks/{id}` - Delete a task
- `GET /docs` - List documents
- `POST /webhooks` - Register webhook endpoints

**Documentation:** Full API docs at https://developers.techcorp.com

### Webhooks
**Get real-time notifications for events:**
- Task created/updated/deleted
- Comment added
- Member joined/left
- Due date approaching

**Setup:**
1. Settings → Developer → Webhooks
2. Click "Add Webhook"
3. Enter your endpoint URL
4. Select events to subscribe to
5. Save and test

## 11. Troubleshooting Guide

### Common Issues

**"I can't log in after password reset"**
- Check spam/junk folder for the reset email
- Ensure you're clicking the link within 24 hours (links expire)
- Try requesting a new reset link
- If still not received, contact support with your email address

**"GitHub integration shows authorization error"**
- Verify you have admin access to the GitHub organization
- Check that third-party app access is enabled in GitHub settings
- Disconnect and reconnect the integration
- Ensure the TechCorp app has access to the specific repositories

**"App crashes when uploading files"**
- Check file size (max 100 MB per file on Pro, 500 MB on Business)
- Ensure stable internet connection
- Try uploading via web browser instead
- Update to the latest app version

**"AI summaries not generating"**
- Check your credit balance (Settings → Billing)
- Free plan users: credits reset on the 1st of each month
- Ensure the document has enough content to summarize (min 100 words)
